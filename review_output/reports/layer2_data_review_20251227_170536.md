### **[CRITICAL] SQL Injection via Dynamic Query Construction**
- **Location**: src/12_processing.py:45-65, src/02_models.py:multiple locations
- **Description**: The code constructs SQL queries dynamically without proper parameterization, particularly in filtering operations.
- **Impact**: Attackers could execute arbitrary SQL commands, leading to complete database compromise, data exfiltration, or data destruction.
- **Proof of Concept**: 
```python
# In process_pending_raw_emails, if mail_account.id is controlled:
# mail_account.id = "1; DROP TABLE users; --"

```
- **Recommendation**: 
```python
# SECURE: Use parameterized queries

# Or ensure all inputs are validated:

```

- **Impact**: Data corruption, partial updates, or inconsistent database state.
